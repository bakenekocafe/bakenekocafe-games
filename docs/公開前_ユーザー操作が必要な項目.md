## 1. api.bakenekocafe.studio を 200 にする

**操作先リンク**
- Cloudflare ダッシュボード: https://dash.cloudflare.com/
  - **Workers** → `bakeneko-api` → Settings → Domains & Routes（Target をコピー）
  - **DNS** → `bakenekocafe.studio` のゾーン → DNS Records（CNAME 追加）

**やること**
1. https://dash.cloudflare.com/ を開く
2. 左メニュー **Workers & Pages** → `bakeneko-api` → **Settings** → **Domains & Routes** を開き、Target（例: `bakeneko-api.○○.workers.dev`）をコピー
3. 左メニュー **Websites** → `bakenekocafe.studio` → **DNS** → **Records** を開く
4. **Add Record** をクリック:
   - Type: **CNAME**
   - Name: **api**
   - Target: 上でコピーした値
5. 保存して数分待つ
6. PowerShell で確認:
   ```powershell
   .\scripts\verify-api-domain.ps1
   ```
7. `[1] DNS: OK` + `[2] HTTP: 200` になれば完了

**参照**: `docs/api.bakenekocafe.studio_チェックリスト.md`

## 1.5. ルートドメイン（www なし）→ `www` へ 301（正規 URL / AdSense）

サイトの正規 URL は **`https://www.bakenekocafe.studio`** のため、**apex**（`https://bakenekocafe.studio`）は **301 で www** に寄せます。  
Cloudflare Pages の `_redirects` ではホスト条件が使えないため、リポジトリ同梱の **Worker**（ルート `bakenekocafe.studio/*` のみ）で処理しています。

### A. Worker で運用する（推奨・本番デプロイ済み）

- コード: `workers/apex-www-redirect/`（名前: **`bakeneko-apex-www-redirect`**）
- 再デプロイ（`wrangler login` 済みの環境で）:
  ```powershell
  cd "c:\Users\yoshi\OneDrive\デスクトップ\KohadaJump_Migration\workers\apex-www-redirect"
  npx wrangler deploy
  ```
- **Workers & Pages** → **Workers** → `bakeneko-apex-www-redirect` でバージョン・ルート `bakenekocafe.studio/*` を確認できる。

### B. ダッシュボードの Redirect Rule で代用する場合

Worker を外し、ゾーンの **Redirect Rules** で同じ挙動にする場合の例（[公式](https://developers.cloudflare.com/rules/url-forwarding/examples/redirect-root-to-www/)）:

- **Request URL**: `https://bakenekocafe.studio/*`
- **Target URL**: `https://www.bakenekocafe.studio/${1}`
- **301**・クエリ維持 **ON**

Worker と **同時に** 同内容のルールを有効にすると二重になりやすいので、どちらか一方にする。

### 確認（PowerShell）

```powershell
cd "c:\Users\yoshi\OneDrive\デスクトップ\KohadaJump_Migration"
.\scripts\verify-www-canonical.ps1
```

- apex が **301** かつ `Location` に **`www.bakenekocafe.studio`** が含まれれば OK。

### Google AdSense

- サイトの登録 URL は **`https://www.bakenekocafe.studio/`** に合わせる（申請画面・サイト一覧）。

- Cloudflare Pages: https://dash.cloudflare.com/ → Workers & Pages

## 5. 実広告SDK 実機テスト

**操作先リンク**
- 編集するファイル: `index.html`（プロジェクトルート直下）
- テスト用URL: https://www.bakenekocafe.studio （デプロイ後）
- Google Ad Placement API ドキュメント: https://developers.google.com/ad-placement/

**やること**
1. `index.html` を開き、Ad Placement API の `<script>` タグの **コメントを外す**
   ```html
   <!-- 本番時にコメントを外す ↓ -->
   <script async
     src="https://pagead2.googlesyndication.com/pagead/js/adsbygoogle.js?client=ca-pub-5189561379756793"
     crossorigin="anonymous"
     data-adbreak-test="on"></script>
   ```
2. 同じく `index.html` 内の設定を変更:
   ```js
   window.BAKENEKO_USE_REAL_ADS = true;   // false → true に
   ```
3. デプロイ後、実機（スマホ or PC ブラウザ）で https://www.bakenekocafe.studio を開く
4. ゲームをプレイ → リザルト画面で「広告を見てこはだを応援する」を押す
5. 広告が表示され、完視聴 or スキップ後に画面が戻れば OK
6. **本番公開時**: `data-adbreak-test="on"` を削除する

**参照**: `docs/実広告SDK_テスト接続とロールバック.md`

## 6. 広告審査申請

**操作先リンク**
- Google AdSense 申請: https://adsense.google.com/start/
- AdSense ヘルプ（審査要件）: https://support.google.com/adsense/answer/10162

**やること**
1. https://adsense.google.com/start/ を開く
2. Google アカウントでログイン（運営用アカウント）
3. サイトURL に `https://www.bakenekocafe.studio` を入力
4. 「サイトの説明」等を求められたら以下をコピペ:
   ```
   BAKENEKO GAMES は、一般社団法人 新宿租界が運営するブラウザゲームサイトです。
   保護猫活動の周知とゲーム体験の提供が目的です。
   ゲームは広告を視聴しなくても最初から最後まで遊べる設計です。
   ```
5. 規約URL等を求められたら以下をそのまま入力:
   - サイトトップ: `https://www.bakenekocafe.studio`
   - 広告の仕組み: `https://www.bakenekocafe.studio/support.html`
   - 利用規約: `https://www.bakenekocafe.studio/terms.html`
   - プライバシーポリシー: `https://www.bakenekocafe.studio/privacy.html`
   - お問い合わせ: `https://www.bakenekocafe.studio/contact.html`
6. 審査を提出 → 結果はメールまたは AdSense ダッシュボードで通知

**参照**: `docs/広告審査申請_手順とチェックリスト.md`

## 6b. bakeneko-cafe.tokyo の ads.txt（公式サイトも AdSense に載せる場合）

**bakeneko-cafe.tokyo** を AdSense のサイト一覧に載せ続ける場合、**そのドメインのルートに `ads.txt` が必要**です（未配置だと ads.txt ステータスが「不明」のままになります）。

**やること**

1. 手順とコピペ用1行は **`docs/bakeneko-cafe.tokyo_ads.txt設置手順.md`** を開く。
2. 同じ内容のファイル **`docs/bakeneko-cafe.tokyo-ads.txt`** を、サーバのドキュメントルートに **`ads.txt`** という名前で配置する。
3. **https://bakeneko-cafe.tokyo/ads.txt**（および `www` を使っているなら **https://www.bakeneko-cafe.tokyo/ads.txt**）がブラウザで開けることを確認する。
4. AdSense のサイト管理で **24〜48時間後** 頃まで様子を見る。

中身は **.studio 用 `ads.txt` と同一**（`pub-5189561379756793`）で正しいです。
