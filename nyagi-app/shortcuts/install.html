NYAGI 音声 → API

ショートカットを取り込み → X-Admin-Key と X-Staff-Id をブロック内で自分用に書き換え → Siri で実行。

いまの QR はネット上の 未署名 定義への取り込みです。iOS 15 以降では拒否されることが多いです。管理者が install.html 内の NYAGI_ICLOUD_SHORTCUT に iCloud 共有 URL を入れて再デプロイすると確実です。それまで NYAGI-音声API-手順.txt の手動作成を使ってください。

① QR をスキャン（推奨）

カメラで下の QR を読み取る。iCloud リンクのときは Safari で「ショートカットを追加」が開きます。

読み取れない場合は ② のボタンをタップ。

② このページから開く ショートカットを取り込む

Safari で開いてタップ。iCloud の場合はそのまま追加画面へ。未設定の場合のみ「ショートカット」アプリの取り込み URL になります。

③ 取り込めなかったとき（必ずできるやり方）

Apple の制限で、ネットから未署名のショートカットを入れられないことがあります。その場合はショートカットアプリでブロックを手動で足すだけで同じことができます（ファイル取り込みは不要）。

  1. 準備：NYAGI と同じ X-Admin-Key と X-Staff-Id（スタッフ ID）をメモ。
  2. 新規ショートカット（＋）→ 名前を「NYAGI 音声送信」などに。
  3. 「テキストを聞き取る」を追加（言語：日本語）。
  4. 「辞書」を追加。キー text、値に「聞き取ったテキスト」。
  5. 「URL のコンテンツを取得」：POST、URL は
    https://api.bakenekocafe.studio/api/ops/voice/shortcut/voice
    ヘッダーに Content-Type: application/json、X-Admin-Key、X-Staff-Id。本文は JSON で、さきほどの辞書ブロック全体。
  6. 「辞書の値を取得」×2：1回目のキー shortcut（入力は URL 取得の結果）→ 2回目のキー speak（入力はその出力）。
  7. 「テキストを話す」に最後の結果を指定（または「結果を表示」）。

画面の説明つきの全文は次のテキストにあります（Safari で開いて共有・コピー可）。

NYAGI-音声API-手順.txt を開く

取り込みに成功した場合も、「URL のコンテンツを取得」内の ここにADMIN_KEYを貼る / ここにスタッフIDを貼る を自分用の値に直してください。
