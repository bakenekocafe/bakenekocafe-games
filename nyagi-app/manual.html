🔑 1. ログイン

NYAGI は現在、4桁パスワードでログインします。スタッフIDを個別入力する方式ではありません。

1index.html か各画面のログインゲートで 4桁パスワードを入力
2「ログイン」をタップ
3認証が通ると、他画面でもそのまま利用できます

認証情報は端末内に保存されるため、通常は次回以降も自動で再利用されます。

💡 明示的にログアウトしたいときは、トップ画面の「ログアウト」を使ってください。

🏠 2. NYAGIトップ

トップ画面は、音声入力とテキスト入力の入口です。

  • 🎤 音声入力: 録音して送信。認識候補が出た場合はその場で選び直せます
  • ⌨️ テキスト入力: 例「こはだ うんちした 普通」のように直接送信できます
  • 入力履歴: 最近送った内容を確認できます
  • 下部ボタン: ダッシュボード / 猫一覧 / スタッフ登録へ移動できます
💡 音声が使えない端末では、案内が「音声非対応」に変わります。その場合はテキスト入力を使ってください。

📊 3. ダッシュボード

全体状況を確認するメイン画面です。拠点とステータスで絞り込んだ内容がまとめて表示されます。

  • 拠点フィルタ: BAKENEKO CAFE / 猫又療養所 / 遠藤宅 / 預かり隊
  • ステータスフィルタ: 在籍 / 卒業 / トライアル中
  • 📈 今日の進捗: タスク・給餌・投薬などの進捗を上部で一覧表示。タップで該当カードへ移動できます
  • アラート: 要注意・経過観察の猫が上部に出ます
  • 異常記録 / 未記録: 当日の気になる記録や未入力項目を確認できます
  • 🏥 健康スコア: フィルタに一致した全猫を、注意順で表示します
  • 📅 明日の予定: 明日対応予定の内容をまとめて確認できます
  • 🏥 病院スケジュール: 直近30日とそれ以降の予定を折りたたみ表示します
💡 拠点・ステータスの選択と、折りたたみ状態の一部は端末側に保存されます。

🎛 4. フィルタ設定

NYAGI では、画面ごとに表示対象を絞り込めます。まずは上部のフィルタを確認すると、必要な情報に早くたどり着けます。

  • ダッシュボード: 拠点 / ステータスで全体表示を切り替えます
  • 猫一覧: 拠点表示と、猫ごと / 項目ごとの見方を切り替えます
  • 掲示板: 拠点 / 猫 / 項目（注意事項・病院記録・病院予定・業務連絡）で絞り込めます
  • タスク: 日付 / 状態 / 拠点 / タブ（今日のタスク・プロジェクト・監視中・テンプレート）で切り替えます
  • フードDB: カテゴリ / 対象動物 / 文字検索で絞り込みます
  • 薬マスター: 対象動物と文字検索で絞り込みます
  • 音声辞書: 拠点で辞書一覧を絞り込みます
  • 引き受け申請: ステータス / フェーズ / 申請者名やメールの文字検索が使えます
💡 ダッシュボードなど一部の画面では、最後に選んだフィルタが端末に記憶されます。表示件数が少ないときは、拠点やステータスが絞られたままになっていないか確認してください。

🐱 5. 猫一覧

猫一覧には、猫ごとと項目ごとの2モードがあります。

  • 猫ごと: 1枚のカードに、その猫のごはん・排泄・投薬・ケア・体重などをまとめて表示
  • 項目ごと: ごはん・排便・排尿・体重・投薬・ケア・医療など、同じ項目を全猫分並べて確認
  • ＋ 新しい猫を登録: 名前・所属・種別を入力して新規登録できます

基本的に、注意が必要な猫が上に来る構成です。

📋 6. 猫詳細

猫詳細は、各猫の情報を折りたたみセクションで管理する画面です。初期状態では閉じている項目が多いので、見たい行をタップして開きます。

  • 🏥 健康スコア
  • 📋 基本情報
  • 📥 引き受け情報 / 🏠 譲渡関連情報
  • 📝 注意事項: 添付付き保存や Slack 送信にも対応
  • ✅ この猫のタスク
  • ⚖️ 体重記録 / 体重推移
  • 🔥 カロリー評価 / 🍽 給餌プラン
  • 🪮 ケア / 🚽 排便 / 💧 排尿 / 🚰 飲水測定
  • 💊 お薬管理 / 📅 病院予定 / 🏥 病院記録
  • 📋 未完了アクション
💡 名前変更時は旧名が音声辞書に自動登録されます。呼び名変更後の音声入力にも反映されます。

🍽 7. ごはん管理

ごはん関連は猫詳細の「🍽 給餌プラン」で管理します。

プランを追加する

1「+ 追加」をタップ
2食事区分・フード・量(g)を入力
3必要に応じて予定時刻やメモを入れて保存

フードは検索しながら選べます。カロリー表示も自動反映されます。

プリセットを使う

1「📋 プリセット」を開く
2一覧から適用、または管理画面で作成・編集

現在は、紐づけプリセットとプリセット管理の両方があります。自動入れ替えの対象を変えたいときは「紐づけ」、中身を変えたいときは「管理」を使います。

あげた / 残し / 手動ログ

  • チェック操作: プラン行のチェックから「あげた記録」を開けます
  • とりあえずあげた: 摂取率を後で入れたいときに使えます
  • 残した量(g)
  • 給餌ログを記録: プラン外の食事も手動追加できます
  • 🎤 音声で記録: 給餌プランの中から直接音声記録できます
💡 カロリー評価には、摂取率が入った給餌ログが反映されます。食べ残しがあったときは残し量まで入力してください。

🩺 8. 体調・ケア記録

体調や日常ケアは、猫詳細の各セクションから個別に入力します。

  • ⚖️ 体重記録: 数値を記録すると、体重推移グラフに反映されます
  • 🚽 排便状況: 健康 / 硬い / 軟便 / 下痢 / 血便などを記録
  • 💧 排尿状況: なし / 少量 / 普通 / 多い / 血尿などを記録
  • 🚰 飲水測定: 対象猫では飲水量の履歴を確認・記録できます
  • 🪮 ケア実施状況: ブラシや処置など、実施 / スキップを残せます
  • 📝 注意事項: 画像・PDF付きで保存でき、必要に応じて Slack にも共有できます
💡 一部の入力欄には 🎤 ボタンがあります。短いメモや本文は音声入力でも補助できます。

💊 9. お薬管理

猫詳細の「💊 お薬管理」で、個別スケジュールと投薬プリセットを扱います。

  • スケジュール: 薬名・用量・単位・頻度・タイミング・投与経路・開始日/終了日を設定
  • 対応頻度: 毎日 / 隔日A・B / 3日に1回 / 週3回 / 週1回 / 月末のみ / 必要時 など
  • 当日ログ: 行ごとに実施・スキップを記録
  • 🔗 紐づけ投薬プリセット: 自動入れ替えに使うプリセットを設定
  • 📋 プリセット: プリセットの中身を作成・編集し、「全て適用」で即時反映
⚠️ プリセット由来の行は、通常の個別編集ではなく「📋 プリセット」側で変更します。

🏥 10. 病院予定・病院記録

通院関連は、予定と記録を分けて管理します。

  • 📅 病院予定: 予定日・予約時刻・病院名・メモを登録。必要なら Slack 通知も可能
  • 🏥 病院記録: 診察内容・病院名・添付ファイルを保存。閲覧モーダルから詳細確認できます
  • 添付対応: 病院記録には PDF / 画像を添付できます
  • ダッシュボード連携: 直近予定はダッシュボードの病院スケジュールにも出ます
💡 記録系は「保存してSlackに送信」ボタンがあるので、登録と共有を一度に済ませられます。

📌 11. 掲示板

掲示板は、業務連絡と猫ごとの注意事項・病院情報をまとめて見る画面です。

  • 業務連絡を投稿: タイトル・本文・添付ファイル・固定設定で投稿
  • フィルタ: 拠点 / 猫 / 項目（注意事項・病院記録・病院予定・業務連絡）で絞り込み
  • 業務連絡カード: 上部に折りたたみ表示。初期状態は閉じています
  • 猫関連タイムライン: 注意事項・病院記録・病院予定を新しい順で表示します
💡 掲示板は「猫ごとのメモ一覧」でもあります。注意事項や病院登録をしたあと、共有確認に使うと便利です。

✅ 12. タスク（概要）

タスク画面は、単なるチェックリストではなく、毎日のお仕事（ルーティン）・イベント・監視・プロジェクトをまとめた管理画面です。ここでは「10歳の子に説明するつもりで」しくみを書きます。

  • 今日のタスクタブ: 選んだ日の「やることリスト」。チェックを付けたり、メモを足したりできます
  • プロジェクトタブ: 大きな仕事を、木の枝のように小分けして管理します
  • 監視中タブ: しばらく様子を見続けたいことを置いておく場所です
  • テンプレート: 「毎朝これを出す」などの型を作り、朝の一括生成で今日の行にコピーします
  • 🔒 業務終了: その日の締めとして、未処理や引き継ぎを確認します
💡 猫に紐づけたタスクは、猫詳細や猫一覧の「今日のタスク」にも出ます。イベントで期限が過ぎていると、猫詳細では目立つ表示になります。

12-1. タスクの種類（ざっくり言うと）

NYAGI の「1行＝1タスク」には、次のタイプがあります。混同しやすいので名前だけ先に覚えてください。

  • ルーティン（通常）…「その日の日付にやること」。開店準備や給餌チェックなど、毎日回る作業が中心です。
  • イベント…「期限」が主役のタスク。イベント会場の設営のように、「いつまでに終わらせるか」が大事なものに使います（後述）。
  • 監視…「様子を見る」系。テンプレで監視タイプにすると専用タブにまとまり、猫の切り替えや状態の記録向きです。

あわせて、タスクには 属性（カテゴリ） があります。開店準備・イベント・猫のお世話・医療・清掃・閉店・プロジェクト・その他、というラベルで、一覧を色分けしたり進捗バーを並べたりするのに使います。属性「プロジェクト」は主に テンプレートから付けられます（12-7 参照）。プロジェクトタブのツリーとは別物ですが、どちらも「大きな仕事のかたまり」に関係するので、木で全体像を作りつつ、今日のタスクではテンプレの属性をプロジェクトに寄せると探しやすくなります。

⚠️ 「イベント属性」と「イベントタイプのタスク」は名前が似ています。属性は分類ラベル、タイプはデータの種類です。

12-2. イベントタスク（使い方）

いつ使う？　「〇月〇日までに片付ける」「〇日のイベントまでに申込みを終える」のように、締め切りがあり、当日だけでなく前からリストに載せておきたいとき向けです。ルーティンは「その日が来たら出る」一方、イベントは期限が来る前から「まだ終わってないよ」と一覧に残ります。

データの考え方（裏側のルール）　システム上、イベント行は 期限日 を deadline_date（なければ due_date）に入れます。ルーティンは「予定日」として scheduled_date を使います。だからテンプレートの種類を「イベント」にすると、自動生成時も期限の置き方が変わります。

  • 未完了・進行中のイベントは、画面上で選んでいる日付が期限と同じか、期限より後のときに出ます。まだ先の未来の期限だけのときは、その日が近づくまで一覧に出ません（予定表がごちゃごちゃにならないため）。
  • 一覧の対象はおおむね今日から数えて約30日先までの範囲でまとめられます（API が「ここまで先のイベントも見るよ」という終わり日を返します）。
  • 完了・スキップ済みのイベントは、「その作業を終えた日」が、見ている日付と一致するときに表示されます（その日の記録として振り返りやすいように）。
  • 期限を過ぎても未完了だと、一覧の上の方に寄せて表示され、経過日数のバッジが付くことがあります。猫詳細でも、期限超過のイベントは強調されます。
1テンプレートで新規作成し、種別で「イベント」を選ぶ
2繰り返し（毎日／曜日／月次など）を設定。イベントでも繰り返しルールは使えます
3朝の「テンプレートから一括生成」で、その日の行ができる。イベントはその日が期限になります
4テンプレ画面の一括生成で、チェック「イベントを指定日に強制生成」を入れると、イベント型テンプレだけ曜日・月次などの繰り返しルールを無視して、その日付の行を作れます（臨時のイベント日に使います）
5終わったら「完了」。やらない日は「スキップ」と理由メモを残すと、あとから状況が分かりやすいです
💡 迷ったら「締切があるか？」で判断。毎日同じ時間にやるだけならルーティン、〇日までに終わらせるならイベント、のイメージです。

12-3. 並び順（ソート）— なぜこの順番？

「今日のタスク」一覧の並びは、コンピュータが次のルールで決めています（上から順に効きます）。

  1. 期限切れのイベントを先頭へ… 今日見ている日付より前の期限で、まだ完了していないイベントは、いちばん上に近づきます。火事場のバケツを先に届けるイメージです。
  2. 日付の早い順… 予定日・期限日・due_date をまとめた「並べる用の日付」が早い行が先です。
  3. 並び順番号（sort_order）… テンプレートやタスクに書いた数字が小さいほど上。同じ日なら「店長が決めた順」として使えます。
  4. 優先度（priority）… urgent（緊急）→ high → normal → low の順で上に来ます。
  5. 猫の名前… まだ同じなら、五十音っぽく猫名で並べ替えます。
  6. 時刻（due_time）… 同じ猫なら、時刻の早い方が上。
  7. タイトル… 最後のおまじないです。

属性グループ（開店・イベント・猫のお世話…）の並びは、アプリ側の決め台詞の順です。画面の進捗バーも、その順で並びます。中身のタスク行は、上のルールで並びます。

💡 「並び順番号」を使うと、店舗ごとの決まり文句（「まず解錠、その次に給餌チェック」など）を崩さず並べ替えられます。優先度は緊急時の上書きに使うと分かりやすいです。

12-4. テンプレートと一括生成

  • テンプレートは「型」です。毎日・特定の曜日・毎月・1回だけなど、繰り返しルールを付けられます。
  • 一括生成を押すと、選んだ日付に合うテンプレートだけがコピーされ、今日のタスクの行が増えます。すでに同じテンプレから同じ日の行があるときはスキップされます。
  • 1回だけ（once）のテンプレは、一度生成されると自動でオフになるので、使い捨てのイベント用にも向きます。
  • 生成が終わったあと、Slack に共有ボタンが出ることがあります。朝礼で「今日これが増えました」と流す用途です。

12-5. 画面のタブの使い分け

  • 今日のタスク… 日付・拠点・担当・状態で絞り込み。属性ごとの進捗バーをタップすると、その属性のところまでスクロールできます。
  • プロジェクト… プロジェクト一覧から開くと、親子のノード（やることの階層）を編集します。大掃除や移転準備のチェックリスト向けです。
  • 監視中… 監視タイプのタスクだけを密度高く一覧します。猫の入れ替えや状態変更が中心の画面です。

12-6. 操作手順（画面の押し方・ケース別）

共通の入口は tasks.html（タスク画面）です。左上の拠点を合わせてから作業してください。

ケース A｜「その日だけ」のイベントを、すぐ手で1件作る（おすすめ）

譲渡会・セール当日など、その日限りで終わらせたい用事向けです。行は消えずDBに残りますが、翌日以降の日付で一覧を見ると出てこないので、運用上は「その日のリストからは消えた」と考えて大丈夫です。

1タブ 今日のタスク を開く
2下の + タスクを追加 をタップ
3タイトルを入力（例: 「4/20 譲渡会ブース片付け」）
4タスク種類で イベント を選ぶ
5指定期日に、その用事の期限日（＝その日だけやりたい日）を入れる。必要なら 指定実行日にも同じ日付を入れると分かりやすいです
6属性・猫・依頼先・優先度は店のルールに合わせて設定（任意）
7追加をタップする
8画面上部の日付を、その期日と同じ日に合わせて 更新。一覧に出たら、その日のうちに 完了 または スキップ
💡 フィルタの状態が 未完了 だけのときは、完了すると一覧から消えて見えます。あとから確認したいときは状態を すべて か 完了 に変えてください。

ケース B｜「1回だけ」のテンプレから、朝に1行だけ生やす（チームで繰り返し使わない）

毎年は不要で、今年の〇月〇日だけ出したいとき向け。一括生成をしたあと、テンプレートが自動でオフになるので「うっかり明日も同じ行が増える」ことがありません。

1タブ テンプレート を開く
2+ 追加 をタップ
3テンプレートID（英字・数字・アンダースコア推奨。例: tmpl_sale_20260420）
4タイトル・タスク種類: イベント・属性などを入力
5繰り返しで 1回のみ を選んで保存
6イベント当日の朝: 生成日をその日に合わせ、⚡ 一括生成 を実行
7タブを 今日のタスク に戻り、同じ日付で更新して行ができているか確認 → 終わったら完了
💡 繰り返しが「曜日指定」などで、当日がルールに合わないテンプレだけ先に作ってあるときは、一括生成の横のチェック イベントを指定日に強制生成 をオンにすると、その日付で無理やり1行作れます。

ケース C｜毎朝同じルーティンを量産する（開店チェックなど）

1テンプレート → + 追加
2タスク種類: ルーティン、繰り返し: 毎日（または曜日指定）
3保存したら、毎朝 一括生成 でその日の行を作る（自動Cronがある運用なら手動は不要な日もあります）

ケース D｜何日か前から「期限が迫ってるよ」と出したいイベント

テンプレまたは手入力で イベント を作り、指定期日を本番日にします。一覧の日付を期限と同じ日か、期限より後にすると、未完了のままだと上の方に出て目立ちます（期限を過ぎた未完了は強調）。

ケース E｜猫を付けた監視タスク

1タブ 監視中 または + 監視タスクを追加
2タスク種類: 監視、監視期限を必ず意識して入力
3猫を選ぶと、猫詳細のタスク欄にも関係が出やすくなります
⚠️ 「完了したらデータベースから物理削除」ではありません。一覧とフィルタで見えなくなるイメージです。監査のため行は残ります。
⚠️ 業務終了は未処理タスクの確認に使います。締めの前に、イベントの期限超過が残っていないかも見ておくと安心です。

12-7. イベントタスクと「プロジェクト」の違い（混同ポイントの整理）

会話で「プロジェクトタスク」と言うと、人によって別のものを指していることがあります。NYAGI では次のように分けると噛み合います。

まず結論（1行ずつ）

  • イベントタスク＝「締切」が軸の1行のタスク（種類は イベント）。今日のタスクに並び、ここで完了／スキップします。
  • プロジェクト（タブ）＝大きな仕事を親子のツリーで分解する別の画面。木のノードを追うイメージです。
  • 属性「📁 プロジェクト」＝進捗バー用の分類ラベル（データ上は project）。タスク種類に「プロジェクト」はありません（種類はルーティン／イベント／監視の3つだけです）。

比べてみる表

比較軸 イベントタスク プロジェクト（タブのツリー）
主な画面 今日のタスク（日付つき一覧） プロジェクトタブ（一覧→詳細の木）
何を表すか 〇日までに片付けるなど、期限が決まった用事の1件 段取り全体（大項目→中項目→小項目）
完了の感じ その1行を完了にすると、その行はルールどおり一覧から見えなくなる（日付・フィルタ次第） ノードを順番にチェック。ノードと「今日のタスク」の行が紐づいている場合は、片方を完了するともう片方も進むことがあります（運用で連携したとき）
向いている例 譲渡会当日チェック、申込締切までの作業、〇日までの片付け 店舗改装、引越し準備、大型イベントの工程表そのもの

属性「📁 プロジェクト」との関係（画面の実装メモ）

テンプレートを追加・編集するときの「属性」には 📁 プロジェクト を選べます。こうして生成された行は、今日のタスクの進捗バーで プロジェクトの山に入ります。一方、画面下の + タスクを追加 モーダルの属性プルダウンは、現状（マニュアル更新時点）では 開店準備〜その他までに限られており、ここから直接「📁 プロジェクト」は選べません。大改装の締切を「プロジェクト山」に載せたいときは、テンプレートで属性をプロジェクトにして一括生成する、などのやり方になります。

種類＝イベント と 属性＝プロジェクト は独立です。テンプレートで両方付ければ、「締切付きの用事」が「プロジェクト山」に並びます。

💡 迷ったら「締切の1行として回したい？」→イベント。「全体の段取り表を木で見たい？」→プロジェクトタブ。両方使うなら、木で全体像＋締切が近いところだけ今日のタスクにイベント行、の二段構えが分かりやすいです。
⚠️ 「イベント属性」（分類ラベル）と「イベント種類のタスク」は別物です。属性は進捗バーの色分け、種類は期限の扱いが変わります（12-1 の注意と同じです）。

🥫 13. フードDB

フードDBは、給餌に使う製品情報を登録・検索する画面です。

  • 製品名 or URL 検索: 自動取得候補から選び、プレビューを確認して登録できます
  • 入力項目: ブランド / 製品名 / カテゴリ / 形態 / 対象動物 / kcal / 栄養成分など
  • 一覧絞り込み: 療法食 / 総合栄養食 / 一般食・おやつ / 猫用 / 犬用
  • 辞書編集: 製品名の表記ゆれ・別名辞書を詳細モーダルから追加・編集できます

💊 14. 薬マスター

薬マスターは、投薬スケジュールの元になる薬データベースです。

  • 薬名 or URL 検索: 候補取得後、プレビューして登録できます
  • 編集項目: 薬名 / 一般名 / カテゴリ / 対象動物 / 剤型 / 単位 / 備考
  • 一覧フィルタ: 猫用 / 犬用 / 共通
  • ＋ ボタン: 手動で新しい薬を追加できます

🎤 15. 音声辞書

音声辞書画面では、猫名の聞き違い対策を行います。

  • 辞書一覧: 拠点ごとに、猫名の別表記・誤変換を確認できます
  • 手動で辞書に追加: よくある誤認識を任意の猫へ紐づけられます
  • 誤認識ログ: 解決できなかった名前トークンを集計表示します
  • ログから辞書へ: ログ行を選び、そのまま辞書登録できます
  • 自動追加: 条件を満たすログをサーバー側ルールで辞書化できます

📥 16. 引き受け申請

intake-admin.html は、スタッフ向けの申請管理画面です。先にNYAGI本体へログインしておく必要があります。

  • 招待リンク発行: 外部申請者向けの setup URL を作成します
  • 申請一覧: draft / submitted / under_review / approved などで絞り込み可能
  • 申請者一覧: フェーズやメール・名前で検索できます
  • 詳細パネル: 選択した申請・申請者の内容を右側で確認します
💡 外部の申請者が使うのは別ページです。この画面は招待・審査をするスタッフ専用です。

🔧 17. トラブルシューティング

画面が古い / 表示がおかしい

1各画面ヘッダー上部の「🔄 キャッシュクリア」をタップ
2必要なら clear-cache.html を開いて再読込

ログインが通らない

  • 4桁パスワードが正しいか確認
  • 一度ログアウトして再ログイン
  • それでもだめならサイトデータ削除後に再試行

音声入力が動かない

  • ブラウザのマイク許可を確認
  • Safari / Chrome 系を推奨
  • 非対応表示のときはテキスト入力へ切り替え

添付できない

  • 病院記録・注意事項は主に画像/PDF、掲示板は画像やOffice系も対応
  • 通信が不安定なときは、添付を減らして再送してください
